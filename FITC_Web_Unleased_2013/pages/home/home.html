First-Class Citizens

HTML/CSS/JS are now first-class citizens for app development

(Colloquially, they are called WinJS apps)

C#, C++, and VB.NET are peers

And Why Not?

  • A DOM is a DOM
  • JS performance is near parity with compiled languages
  • Well known and well documented
  • Huge open source community
  • Unified platform
  • All Windows Store apps have access to the same resources

The Rough Spots

  • The testability story is pretty poor
  • Best practices are still being developed
  • Still need to watch out for JS pitfalls

The Opportunity

  • Half a billion Windows 8 compatible machines
  • An uncongested store

Anatomy of a WinJS App

WinJS apps are very similar to single page web apps

WinRT exposes system services similarly to how node.js does

Root Page


<body>
  <div id="contenthost"
       data-win-control="Application.PageControlNavigator"
       data-win-options="{home: '/pages/home/home.html'}"></div>
</body>

We've encountered our first control.

Controls in jQuery UI


<div id="progressbar"></div>
					
<script>
	$(function() {
		$( "#progressbar" ).progressbar({
			value: 37
		});
	});
</script>

Lots going on behind the hood.

Controls in WinJS


<progress id="determinateProgressBar" value="37" max="100"></progress>

That's sort of cheating, but not really.

Complex Controls in WinJS


<div id="ratingControl"
     data-win-control="WinJS.UI.Rating"
     data-win-options="{averageRating : 9, maxRating : 10}"></div>
					
<script>
    WinJS.UI.processAll(rootElement);
</script>
